[00:00 – 00:30]  THE HOOK
--------------------------
Open with a question directed at the audience — no slides yet:

  "How many of you have generated a Net Worth certificate this month?
   How long did it take?"

Wait 2 seconds. Then:

[00:30 – 01:30]  THE PROBLEM (make them feel it)
-------------------------------------------------
- 3.5 lakh+ CAs in India. Certificate generation is daily, repetitive,
  high-stakes work.
- Manual flow: open balance sheet → find the right figures → pick the correct
  NW method → type the certificate → review → sign → send.
  That's 30–45 minutes minimum.
- Error risk: wrong method for entity type (LLP vs Company), misread figures,
  wrong financial year — all lead to professional liability.
- No existing tool addresses this end-to-end for ICAI-format certificates.

KEY LINE:
  "The problem isn't that CAs are slow.
   It's that a task this repetitive has no business being manual in 2025."

--------------------------------------------------------------------------------

[01:30 – 07:00]  THE LIVE DEMO  ← THIS IS YOUR SCORE
------------------------------------------------------
Run in this exact sequence — each step reveals a new differentiator.

  STEP 1 — Upload (30 sec)
  Drop in a real-looking balance sheet PDF.
  Say: "PDF or Excel — doesn't matter."

  STEP 2 — Redaction prompt appears (45 sec)
  "Before this document touches the AI, we auto-redact CIN, PAN, GSTIN,
   Aadhaar, and mobile numbers. The AI never sees raw client data.
   This is our privacy-first architecture — and this alone sets us apart
   from every generic AI tool."
  Show the manual draw-box feature for 10 seconds.
  "If there's anything else sensitive, the CA draws a box over it."

  STEP 3 — AI extracts (45 sec)
  While it loads, say:
  "Claude AI is reading the redacted document right now, extracting figures
   with source citations."
  When results appear, point to the citation trail:
  "Every figure is linked back to the exact line it came from.
   Full audit trail — essential for professional accountability."

  STEP 4 — Certificate type + Entity + Method (45 sec)
  Select Net Worth → select Company → show the 7 method cards.
  "This is where most tools fail. Net Worth means different things for an LLP
   vs a Company vs a SEBI-listed entity. We implement 7 calculation methods —
   Book Value, Tangible NW, Companies Act method, SEBI LODR, and more.
   The AI recommends the right one."

  STEP 5 — Live preview (30 sec)
  "Edit values, change narrative, add UDIN, set a PDF password —
   and the certificate updates in real time on the right."

  STEP 6 — Download + Email (30 sec)
  Click Generate. Show PDF download + DOCX + Email to Client button.
  "PDF and Word document. One click to send to the client.
   Certificate number auto-generated, history saved."

  STEP 7 — YoY / Ratio Analysis (15 sec)
  "Bonus — year-on-year comparison and ratio analysis are built in
   for the CA who wants to go deeper."

--------------------------------------------------------------------------------

[07:00 – 08:00]  TECHNOLOGY (hit these 3 points)
-------------------------------------------------
1. PRIVACY-FIRST PIPELINE
   Redact PII → send clean doc to Claude → extract → generate.
   Data never stored after session.

2. CLAUDE SONNET 4.6 WITH STRUCTURED EXTRACTION
   Not a chat wrapper. Custom prompts per certificate type, per entity type,
   per NW method. Vision fallback for scanned PDFs.

3. STACK
   FastAPI backend, React + TypeScript frontend, python-docx for ICAI-format
   output, no database — zero setup, zero data risk.

KEY LINE:
  "We didn't build a chatbot.
   We built a domain-specific workflow engine
   that happens to use AI for the hard part."
